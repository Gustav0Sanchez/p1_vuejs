Glosario

Este glosario tiene como objetivo proporcionar orientación sobre los significados de los términos técnicos que se utilizan habitualmente cuando se habla de Vue. Su objetivo es describir cómo se utilizan habitualmente los términos, no es una especificación prescriptiva de cómo deben utilizarse. Algunos términos pueden tener significados o matices ligeramente diferentes según el contexto en el que se encuentren.

1. API de composición:

La API de composición es una colección de funciones que se utilizan para escribir componentes y elementos componibles en Vue.
El término también se utiliza para describir uno de los dos estilos principales que se utilizan para escribir componentes; el otro es la API de opciones. Los componentes escritos con la API de composición utilizan una función <script setup> explícita .setup().

2. API de reactividad:

La API de reactividad es una colección de funciones básicas de Vue relacionadas con la reactividad. Estas se pueden usar independientemente de los componentes. Incluye funciones como ref(), reactive(), computed(), watch()y watchEffect().
La API de reactividad es un subconjunto de la API de composición.

3. Componente:

El término componente no es exclusivo de Vue. Es común en muchos marcos de interfaz de usuario. Describe una parte de la interfaz de usuario, como un botón o una casilla de verificación. Los componentes también se pueden combinar para formar componentes más grandes.
Los componentes son el mecanismo principal proporcionado por Vue para dividir una interfaz de usuario en partes más pequeñas, tanto para mejorar la capacidad de mantenimiento como para permitir la reutilización del código.
Un componente Vue es un objeto. Todas las propiedades son opcionales, pero se requiere una plantilla o una función de renderizado para que el componente se renderice.

4. Componente asincrónico:

Un componente asincrónico es un contenedor alrededor de otro componente que permite que el componente contenedor se cargue de forma diferida. Esto se utiliza normalmente como una forma de reducir el tamaño de los .jsarchivos compilados, lo que permite dividirlos en fragmentos más pequeños que se cargan solo cuando es necesario.
Vue Router tiene una característica similar para la carga diferida de componentes de ruta, aunque no utiliza la característica de componentes asíncronos de Vue.

5. Componente dinámico:

El término componente dinámico se utiliza para describir casos en los que la elección de qué componente secundario se va a representar debe realizarse de forma dinámica. Normalmente, esto se logra mediante <component :is="type">
Un componente dinámico no es un tipo especial de componente. Cualquier componente puede utilizarse como componente dinámico. Lo que es dinámico es la elección del componente, no el componente en sí.

6. Componente funcional:

Una definición de componente suele ser un objeto que contiene opciones. Puede que no parezca así si estás usando <script setup>, pero el componente exportado desde el .vue archivo seguirá siendo un objeto.
Un componente funcional es una forma alternativa de componente que se declara mediante una función. Esa función actúa como función de representación del componente.
Un componente funcional no puede tener ningún estado propio. Tampoco pasa por el ciclo de vida habitual de los componentes, por lo que no se pueden utilizar ganchos de ciclo de vida. Esto los hace ligeramente más livianos que los componentes normales con estado.

7. Componente web:

El estándar de componentes web es una colección de características implementadas en los navegadores web modernos.
Los componentes de Vue no son componentes web, pero defineCustomElement() se pueden utilizar para crear un elemento personalizado a partir de un componente de Vue. Vue también admite el uso de elementos personalizados dentro de los componentes de Vue.

8. Componible:

El término "componible" describe un patrón de uso común en Vue. No es una característica independiente de Vue, es solo una forma de usar la API de composición del marco.

9. Directiva:

El término directiva se refiere a los atributos de plantilla que comienzan con el v-prefijo o sus abreviaturas equivalentes.
Las directivas integradas incluyen v-if, v-for, v-bind, v-ony v-slot.
Vue también admite la creación de directivas personalizadas, aunque normalmente solo se utilizan como una "vía de escape" para manipular nodos DOM directamente. Las directivas personalizadas generalmente no se pueden utilizar para recrear la funcionalidad de las directivas integradas.

10. DOM Virtual:

El término DOM virtual (VDOM) no es exclusivo de Vue. Es un enfoque común que utilizan varios marcos web para gestionar actualizaciones de la interfaz de usuario.
Los navegadores utilizan un árbol de nodos para representar el estado actual de la página. Ese árbol y las API de JavaScript que se utilizan para interactuar con él se denominan modelo de objetos de documento o DOM.
La manipulación del DOM es un importante obstáculo para el rendimiento. El DOM virtual ofrece una estrategia para gestionarlo. En lugar de crear nodos DOM directamente, los componentes Vue generan una descripción de los nodos DOM que desean. Estos descriptores son objetos JavaScript simples, conocidos como VNodes (nodos DOM virtuales). La creación de VNodes es relativamente económica.

11. Ganchos de ciclo de vida:

Una instancia de componente de Vue pasa por un ciclo de vida. Por ejemplo, se crea, se monta, se actualiza y se desmonta.
Los ganchos del ciclo de vida son una forma de escuchar estos eventos del ciclo de vida
Con la API de opciones, cada gancho se proporciona como una opción independiente, por ejemplo mounted. La API de composición utiliza funciones, como onMounted().

12. Izamiento:

El término "elevarse " se utiliza para describir la ejecución de una sección de código antes de alcanzarla, antes que de otro código. La ejecución se "eleva" a un punto anterior.
var JavaScript utiliza elevación para algunas construcciones , como importy declaraciones de funciones.
En un contexto de Vue, el compilador de plantillas aplica elevación estática para mejorar el rendimiento. Al convertir una plantilla en una función de renderización, los VNodes que corresponden a contenido estático se pueden crear una vez y luego reutilizar. Estos VNodes estáticos se describen como elevados porque se crean fuera de la función de renderización, antes de que se ejecute. Una forma similar de elevación se aplica a los objetos estáticos o matrices que genera el compilador de plantillas.

13. Nodo Virtual:

Un VNode es un nodo DOM virtual . Se pueden crear utilizando la h() función.

14. Ranura:

Los slots se utilizan para pasar contenido a los componentes secundarios. Mientras que los props se utilizan para pasar valores de datos, los slots se utilizan para pasar contenido más completo que consta de elementos HTML y otros componentes de Vue.
